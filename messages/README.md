## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Installation

```bash
$ npm install
```

## Running the app

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

## Test

```bash
# unit tests
$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```

### file の自動生成

- module の生成：`nest generate module <module_name>`
- controller の生成: `nest g controller <folder>/<Call the class> --flat` --flat means don't create an extra folder called 'controllers'

### Data transfer object

- Carries data between two places
- request -> dto -> reueust handler

### validation pipe

1. Use class-transformer to trun the body into an instance of the Dto class
2. Use class-validator to validate the instance
3. If there are validation errors, respond immediately, otherwise provide body to request handler

### Services and repositories

| Services                                             | Repositories                                                          |
| ---------------------------------------------------- | --------------------------------------------------------------------- |
| It's a class                                         | It's a class                                                          |
| place to put any business logic                      | place to put storage-related logic                                    |
| Uses one or more repositrories to find or store data | Usually ends up being a TypeORM entity, a Mongoose schema, or similar |

### Dependency Injection

`Inversion of Control Principle`: Classes should not create instances of its dependencies on its own

1. At startup, register all classes with the container
2. container will figure out what each dependency each class has
3. We then ask the container to create an isntance of class for us
4. Container creates all required dependencies and gives us the instance
5. Container will hold onto the created dependency instances and reuse them if needed

- 1 ~ 2 の手順で、それぞれのクラスに `Injectable` decorator を使い、プロバイダーのモジュールリストに加える
- 3 ~ 4 で Nest が自動的にコントローラーインスタンスを作る
- DI が test が楽になる
